# AGENTS.md

このリポジトリで AI エージェントが作業する際のルールを定義する。

---

## リポジトリ概要

LT（Lightning Talk）発表資料を管理するリポジトリ。
資料は Markdown 形式で作成し、スライドツール（Marp 等）で変換して使用することを想定する。

```
.
├── .agents/                  # エージェント向けルール・指示ファイル
│   ├── AGENTS.md             # 本ファイル（総合ルール）
│   └── lt_nagoya_castle_datacenter.md  # LT個別指示
├── images/                   # 地図・図版画像
├── scripts/                  # 画像生成スクリプト
├── lt_nagoya_castle_datacenter.md  # LT本文（Markdown）
├── AGENTS.md -> .agents/AGENTS.md  # シンボリックリンク
└── CLAUDE.md -> .agents/AGENTS.md  # シンボリックリンク
```

---

## 一般的な安全ルール

### Git 操作

- **main ブランチへの直接 push は禁止**。必ず feature ブランチで作業し PR 経由でマージする
- 作業ブランチ名は `claude/<topic>-<suffix>` 形式とする（既存ブランチがある場合はそちらを使用）
- コミットは意味のある単位でまとめ、コミットメッセージは日本語・英語どちらでも可
- **ファイルの削除・大幅な上書き**は、操作前にユーザーに確認する
- `git push --force` は使用しない

### ファイル操作

- 既存ファイルを編集する前に必ず内容を読み込む
- 一時ファイル・デバッグ用ファイルはコミットしない（`.gitignore` を活用）
- 画像・バイナリファイルは `images/` に配置する

### 外部サービス・ネットワーク

- 地理院タイル・OSM タイルへのアクセスはサンドボックス環境でブロックされる場合がある
  → その場合は `scripts/generate_maps.py` による PIL 模式図で代替する
- 外部 URL を資料に記載する場合は、出典・アクセス日を明示する

### 機密情報

- 個人情報・認証情報（トークン、パスワード等）をファイルにコミットしない
- 発表者の氏名・所属は `（名前）` `（イベント名）` のようにプレースホルダーで残す

## 資料作成ルール

### 言語・形式

- **本文は日本語**で記述する
- スライド区切りは `---`（水平線）を使用する
- 見出しレベル：`#` タイトル / `##` スライド / `###` スライド内小見出し

### 事実確認・引用

- 地質・歴史・技術的な数値・固有名詞は**出典を確認してから記述**する
- 引用は以下の形式で記載する：
  ```
  > 著者（所属）「論文タイトル」
  > 発表先・年
  > URL
  ```
- 推定・概算値を使う場合は「約〜」「概算」と明示し、出典がない場合はプレースホルダーを残す

### 地図・画像

- 地図画像は `images/` ディレクトリに配置し、Markdown から相対パスで参照する
- 地理院タイルが使えない場合は `scripts/generate_maps.py` を実行して模式図を生成する
- 画像には必ず出典・作図根拠をキャプションまたはスライド本文に記載する

### 内容の修正

- 内容に誤りが見つかった場合は、修正範囲を明示してからファイルを更新する
- 固有名詞（会社名・施設名・地名）は特に慎重に確認する
